# Data Model: Guided Mediation (Phase 3)

**Date**: 2026-04-17
**Spec**: [spec.md](spec.md)
**Plan**: [plan.md](plan.md)

Phase 3 is additive. The Phase 1/2 schema (`disputes`,
`notifications`, `dispute_state_transitions`, `schema_version`) is
unchanged and remains the source of truth for dispute detection and
solver notification. Phase 3 introduces a new migration (v3) adding
the tables below.

## SQLite Schema

### Phase 3 Tables

#### mediation_sessions

One row per mediation attempt for a given dispute. A dispute may
have multiple mediation sessions over time (e.g. if the dispute is
re-disputed after a prior summary).

A row in this table represents a session whose `TakeDispute` action
has already succeeded for Serbero (FR-122). A failed start attempt
MUST NOT produce a row here — the attempt is recorded in
`mediation_events` only (see `start_attempt_*` event kinds). This
preserves the invariant "every session row has a successful take
behind it", which the FR-124 external-resolution report and every
audit query rely on.

| Column                      | Type    | Constraints                                      | Description                                                                 |
|-----------------------------|---------|--------------------------------------------------|-----------------------------------------------------------------------------|
| session_id                  | TEXT    | PRIMARY KEY                                      | UUID v4 generated by Serbero                                                |
| dispute_id                  | TEXT    | NOT NULL, FK → disputes(dispute_id)              | The Phase 1/2 dispute this session mediates                                 |
| state                       | TEXT    | NOT NULL                                         | See `State Machine` below                                                   |
| round_count                 | INTEGER | NOT NULL DEFAULT 0                               | Completed mediation rounds (outbound+inbound pair)                          |
| prompt_bundle_id            | TEXT    | NOT NULL                                         | Human-readable bundle id (e.g. `phase3-default`)                            |
| policy_hash                 | TEXT    | NOT NULL                                         | SHA-256 hash of the loaded prompt bundle at session-open (see research R-105) |
| instructions_version        | TEXT    | NULL                                             | Optional operator-provided tag for the bundle (e.g. git rev)                |
| assigned_solver             | TEXT    | NULL                                             | Mirror of `disputes.assigned_solver` at last routing decision               |
| current_classification      | TEXT    | NULL                                             | Latest classification label returned by the reasoning provider              |
| current_confidence          | REAL    | NULL                                             | Latest confidence (0.0–1.0)                                                 |
| buyer_shared_pubkey         | TEXT    | NULL                                             | Hex pubkey reconstructed from take-flow material (addressing buyer chat)    |
| seller_shared_pubkey        | TEXT    | NULL                                             | Hex pubkey reconstructed from take-flow material (addressing seller chat)   |
| buyer_last_seen_inner_ts    | INTEGER | NULL                                             | Inner-event `created_at` of last ingested buyer message                     |
| seller_last_seen_inner_ts   | INTEGER | NULL                                             | Same for seller                                                             |
| started_at                  | INTEGER | NOT NULL                                         | Unix seconds at session open                                                |
| last_transition_at          | INTEGER | NOT NULL                                         | Unix seconds at most recent state transition                                |

Indexes: `idx_mediation_sessions_dispute_id` on `dispute_id`; `idx_mediation_sessions_state` on `state` (used by the scheduler for open-session scans).

#### mediation_messages

One row per chat message exchanged during a session.

| Column                 | Type    | Constraints                                              | Description                                                             |
|------------------------|---------|----------------------------------------------------------|-------------------------------------------------------------------------|
| id                     | INTEGER | PRIMARY KEY AUTOINCREMENT                                | Row id                                                                  |
| session_id             | TEXT    | NOT NULL, FK → mediation_sessions(session_id)            |                                                                         |
| direction              | TEXT    | NOT NULL                                                 | `outbound` or `inbound`                                                 |
| party                  | TEXT    | NOT NULL                                                 | `buyer` / `seller` / `serbero` (outbound only)                          |
| shared_pubkey          | TEXT    | NOT NULL                                                 | Shared pubkey used for addressing                                       |
| inner_event_id         | TEXT    | NOT NULL                                                 | Authoritative identity of the inner event (dedup key for inbound)       |
| inner_event_created_at | INTEGER | NOT NULL                                                 | Inner-event `created_at` (authoritative timestamp)                      |
| outer_event_id         | TEXT    | NULL                                                     | Outer gift-wrap id (outbound only, best-effort tracking)                |
| content                | TEXT    | NOT NULL                                                 | Decrypted inner-event content                                           |
| prompt_bundle_id       | TEXT    | NULL                                                     | Bundle that produced this outbound message (outbound only)              |
| policy_hash            | TEXT    | NULL                                                     | Bundle hash that produced it (outbound only)                            |
| persisted_at           | INTEGER | NOT NULL                                                 | Unix seconds when Serbero stored the row                                |
| stale                  | INTEGER | NOT NULL DEFAULT 0                                       | 1 iff inner-event timestamp predated the session's last-seen marker     |

Unique index: `uq_mediation_messages_inner_event` on `(session_id, inner_event_id)` — inbound dedup primary mechanism. `outbound` rows also populate `inner_event_id` (Serbero signs the inner event with the shared-key material, so it has an id) to share the same dedup space.

Indexes: `idx_mediation_messages_session` on `session_id`; `idx_mediation_messages_direction` on `direction`.

#### mediation_events

Audit log of session-level events: transitions, escalation triggers,
timeout firings, reasoning failures, authorization losses, auth-retry
events.

| Column              | Type    | Constraints                                        | Description                                                             |
|---------------------|---------|----------------------------------------------------|-------------------------------------------------------------------------|
| id                  | INTEGER | PRIMARY KEY AUTOINCREMENT                          | Row id                                                                  |
| session_id          | TEXT    | NULL, FK → mediation_sessions(session_id)          | NULL for daemon-level auth-retry events                                 |
| kind                | TEXT    | NOT NULL                                           | Event kind (enumerated below)                                           |
| payload_json        | TEXT    | NOT NULL DEFAULT '{}'                              | Structured payload (small; no transcripts, no rationales)               |
| rationale_id        | TEXT    | NULL, FK → reasoning_rationales(rationale_id)      | Reference into the audit store when relevant                            |
| prompt_bundle_id    | TEXT    | NULL                                               | Bundle active at event time                                             |
| policy_hash         | TEXT    | NULL                                               | Bundle hash active at event time                                        |
| occurred_at         | INTEGER | NOT NULL                                           | Unix seconds                                                            |

Enumerated `kind` values:

| Kind                            | Meaning                                                                                            |
|---------------------------------|----------------------------------------------------------------------------------------------------|
| `start_attempt_started`         | Event-driven start flow (FR-121) began for a dispute (payload: `{dispute_id, trigger: "detected" \| "tick_retry"}`). No session row yet. |
| `start_attempt_stopped`         | A start attempt stopped before `TakeDispute` (payload: `{dispute_id, stop_reason: "ineligible" \| "auth_pending" \| "auth_terminated" \| "reasoning_unhealthy" \| "reasoning_verdict_negative" \| "reasoning_provider_error" \| "policy_escalate"}`). No session row is created. `auth_pending` / `auth_terminated` come from the `AuthRetryHandle` gate (Serbero is not currently authorized as a solver); `policy_escalate` is written by the T104d opening-path escalation branch when the reasoning verdict says escalate. |
| `reasoning_verdict`             | Reasoning produced a verdict during a start attempt (payload: `{dispute_id, verdict: "mediation_eligible" \| "not_eligible", classification, confidence}`; rationale in audit store). Precedes `TakeDispute`. |
| `take_dispute_issued`           | Serbero issued the `TakeDispute` Mostro action for the dispute (payload: `{dispute_id, outcome: "success" \| "failure", reason?}`). The `session_opened` row below is only written on success. |
| `session_opened`                | New `mediation_sessions` row created. MUST be preceded by a successful `take_dispute_issued` for the same dispute within the same attempt (FR-122). |
| `outbound_sent`                 | An outbound chat message was successfully published.                                               |
| `inbound_ingested`              | A first-time inbound event was decrypted, verified, and persisted.                                 |
| `state_transition`              | Session state changed (payload: `{from, to, reason}`).                                             |
| `classification_produced`       | Reasoning provider returned a classification (rationale stored in audit; referenced here).         |
| `summary_generated`             | A cooperative summary was produced (linked to a `mediation_summaries` row).                        |
| `escalation_recommended`        | Session transitioned to `escalation_recommended` (payload: `{trigger, evidence_refs}`).            |
| `handoff_prepared`              | Phase 4 handoff package was assembled (payload: `{handoff_ref}`).                                  |
| `reasoning_call_failed`         | A reasoning call exhausted its retries (payload: `{provider, model, attempt_count, reason}`).      |
| `authorization_lost`            | Mostro rejected / revoked solver authorization mid-session.                                        |
| `auth_retry_attempt`            | Background auth-retry loop performed an attempt (payload: `{attempt, outcome}`).                   |
| `auth_retry_terminated`         | Auth-retry loop hit its terminal cap. One WARN alert is emitted alongside.                         |
| `auth_retry_recovered`          | Auth-retry succeeded; mediation re-enabled (payload: `{after_attempts, after_seconds}`).           |
| `superseded_by_human`           | A human solver took the dispute via Mostro; session closes without further party messages.         |
| `resolved_externally_reported`  | FR-124 final solver-facing report was emitted after the dispute transitioned to a resolved terminal state while Serbero had collected mediation context (payload: `{dispute_id, final_dispute_status, outbound_party_messages_count, had_classification, had_escalation_recommendation, notifier_route: "targeted" \| "broadcast"}`). Fires at most once per dispute. |
| `session_closed`                | Terminal state recorded for the session.                                                           |

Index: `idx_mediation_events_session_kind` on `(session_id, kind)`.

#### reasoning_rationales

Controlled audit store for full reasoning rationales (FR-120). Never
read by normal application logs; rows referenced by `rationale_id`
elsewhere.

| Column       | Type    | Constraints                           | Description                                                                   |
|--------------|---------|---------------------------------------|-------------------------------------------------------------------------------|
| rationale_id | TEXT    | PRIMARY KEY                           | SHA-256 (hex) over `rationale_text` — content-addressed, dedup-friendly       |
| session_id   | TEXT    | NULL, FK → mediation_sessions(session_id) | NULL for daemon-scoped rationales (e.g. classification during handoff prep) |
| provider     | TEXT    | NOT NULL                              | `openai` / `openai-compatible` / `anthropic` / ...                            |
| model        | TEXT    | NOT NULL                              | Model identifier as returned by the provider                                  |
| prompt_bundle_id | TEXT| NOT NULL                              | Bundle that generated this rationale                                          |
| policy_hash  | TEXT    | NOT NULL                              | Bundle hash that generated it                                                 |
| rationale_text | TEXT  | NOT NULL                              | Full unredacted rationale                                                     |
| generated_at | INTEGER | NOT NULL                              | Unix seconds                                                                  |

Operationally: filesystem permissions on the SQLite file protect this
table; Phase 3 does not add an additional encryption layer. See
research R-107.

### Disputes table — Phase 3 extension (optional)

No new columns on `disputes` are *required* for Phase 3: all
mediation-specific state lives in `mediation_sessions`. The existing
Phase 2 columns (`lifecycle_state`, `assigned_solver`,
`last_notified_at`, `last_state_change`) are sufficient for the
Solver-Facing Routing rule.

**Eligibility (FR-123) is computed, not stored.** Mediation eligibility
for a dispute is derived at read time from a composed predicate over
`disputes.lifecycle_state` (must be a non-resolved, non-terminal
state), `mediation_sessions.state` for that dispute (none may be
active or `escalation_recommended`), and `disputes.assigned_solver`
(must not be a human solver that superseded Serbero). Eligibility
MUST NOT be materialized as a single column like
`disputes.mediation_eligible`, and MUST NOT be implemented as
`lifecycle_state = 'notified'` alone. Both the event-driven start
path (FR-121) and the resumption/retry tick (plan.md "Flow:
Resumption and retry tick") evaluate the same predicate.

If, during implementation, we find we need an
`active_mediation_session_id` shortcut on `disputes` to avoid a join
in the re-notification timer, that is the only addition worth
considering and can land as part of migration v3 without further
spec amendment. It is not assumed here.

### mediation_events — dispute-scoped rows

Most `mediation_events` rows are session-scoped. Some event kinds,
however, occur before a session exists or after all sessions for the
dispute have closed:

- `start_attempt_started`, `start_attempt_stopped`, `reasoning_verdict`
  (negative), `take_dispute_issued` (failure): no session row yet for
  this attempt — the `session_id` column is NULL; the payload carries
  `dispute_id` so audit queries can join through `disputes`.
- `resolved_externally_reported`: a session row MAY exist (closed or
  `escalation_recommended`) or MAY NOT exist (take failed, no session
  was committed). When present, `session_id` references it; when
  absent, `session_id` is NULL and the payload's `dispute_id` is the
  only link.

Audit queries that want "everything Serbero did for dispute X" MUST
union the session-scoped rows with the dispute-scoped rows via
`dispute_id` in the payload.

## State Machine — MediationSession

```text
                     ┌─────────────┐
   session open ───▶ │   opening   │
                     └──────┬──────┘
                            │ first outbound sent
                            ▼
                     ┌──────────────────┐
                ┌──▶ │ awaiting_response│ ◀─── another round started
                │    └──────┬───────────┘
                │           │ inbound ingested + classified
                │           ▼
                │    ┌──────────────┐
                │    │  classified  │
                │    └──────┬───────┘
                │           │ policy decision
                │           │
  round_count < max  │         │       cooperative converged
  and not converged  │         │       ┌────────────────┐
                │           │         │                │
                │           ▼         ▼                │
                │    ┌──────────┐  ┌────────────────┐  │
                └────│ follow_up│  │ summary_pending│──┘
                     │ pending  │  └────────┬───────┘
                     └──────────┘           │ summary produced
                                            ▼
                                    ┌──────────────────────┐
                                    │ summary_delivered    │
                                    └──────────┬───────────┘
                                               │ session closes
                                               ▼
                                    ┌──────────────────────┐
                                    │        closed        │
                                    └──────────────────────┘

   (from any non-terminal state)
    any escalation trigger ───▶ ┌─────────────────────────┐
                                │ escalation_recommended  │ ─▶ closed (after handoff)
                                └─────────────────────────┘

   (from any non-terminal state)
    Phase 2 records assigned_solver != Serbero ──▶ ┌─────────────────────┐
                                                   │ superseded_by_human │ ─▶ closed
                                                   └─────────────────────┘

   (from any non-terminal state, including escalation_recommended)
    Phase 1/2 dispute transitions to a resolved terminal state
    ──▶ the session is closed with outcome resolved_externally_reported
        (FR-124). A mediation_events row of kind
        resolved_externally_reported is written. For disputes with
        mediation context but no session row, the event is written
        with session_id = NULL and the dispute is not modeled here
        (it has no session state machine to advance).
```

- Allowed transitions:
  - `opening → awaiting_response`
  - `awaiting_response → classified`
  - `classified → follow_up_pending → awaiting_response` (next round)
  - `classified → summary_pending → summary_delivered → closed`
  - any non-terminal state → `escalation_recommended → closed`
  - any non-terminal state → `superseded_by_human → closed`
- All other transitions are rejected by `mediation/session.rs`.
- Self-transitions are rejected (this matches the stricter
  `LifecycleState` discipline introduced in Phase 2 review).

## Migration notes (v3)

- Guard migration with `schema_version` as in v1/v2.
- All new tables are created with `CREATE TABLE IF NOT EXISTS` so
  reruns are safe.
- Foreign keys to `disputes` require `PRAGMA foreign_keys = ON`
  (already set in `src/db/mod.rs`).
- The unique index on `mediation_messages(session_id, inner_event_id)`
  is the inbound dedup primary.
- No backfill of Phase 3 data from Phase 1/2 tables is performed.
  Phase 3 starts empty on first run of the new schema.

## Migration notes (v4 — Phase 11 mid-session follow-up loop)

Adds two columns to `mediation_sessions` so the ingest tick can drive
the mid-session evaluator idempotently and track bounded failure.

- `round_count_last_evaluated INTEGER NOT NULL DEFAULT 0` — the
  `round_count` value the last successful `advance_session_round`
  call committed for. The evaluator gates on `round_count >
  round_count_last_evaluated` to avoid re-evaluating the same round
  on every tick (FR-127). Backfilled to `0` for existing rows, which
  forces any in-flight Phase 3 session to be re-evaluated once after
  the daemon restarts — acceptable because the alternative is *no*
  evaluation.

- `consecutive_eval_failures INTEGER NOT NULL DEFAULT 0` — monotonic
  counter of back-to-back reasoning-call or commit failures for the
  mid-session evaluator (FR-130). Incremented on failure, reset to
  `0` on any successful evaluation. At value `3` the session
  escalates with `EscalationTrigger::ReasoningUnavailable` and the
  counter is reset by the escalation path.

Both columns use `ALTER TABLE mediation_sessions ADD COLUMN ...` in
the v4 migration body; SQLite supports adding columns with a default
value and rewrites the page lazily. The migration guard (`schema_version
> 3`) protects re-runs. No data backfill from other tables is
performed.

The `follow_up_pending` state in `mediation_sessions.state` remains
defined for now but is NOT written by the Phase 11 loop; a later
housekeeping PR may remove it.
